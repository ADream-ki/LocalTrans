# Agent Guide

## Mandatory structure (Manual CLI mode)

- Keep CLI parse and branch logic only in `src-tauri/src/main.rs`.
- Keep GUI boot and invoke registration only in `src-tauri/src/lib.rs`.
- Keep all business logic only in `src-tauri/src/commands/` with one file per feature.
- Do not add any `cli` field to `src-tauri/tauri.conf.json`.
- Reuse the same business functions for both CLI branch and Tauri `invoke_handler`.

## Runtime rules

- No argument or subcommand: start GUI.
- With subcommand/flags: execute CLI, print JSON pretty to stdout, and exit `0`.
- On command failure: write structured JSON error to stderr and exit `1`.
- Keep Windows release black-window behavior by preserving:
  `#![cfg_attr(all(not(debug_assertions), target_os = "windows"), windows_subsystem = "windows")]`.

## Implementation notes

- Use `PathBuf` and normalize via `dunce::canonicalize` where path processing is required.
- Keep capabilities minimal; only grant permissions that are used.
